# 剪刘海 · Game Design Document

> 一款 2D 物理操作类小游戏，玩家扮演理发师，操控一把受重力影响的剪刀，按顾客比划的长度剪出整齐的刘海。

---

## 一、游戏概述

### 1.1 高概念
Foddy 风格的轻物理小游戏。难度来源于剪刀的不稳定操控感、玩家自身的记忆任务、以及苛刻的评分系统。

### 1.2 核心体验
- **紧张**：剪刀一直在掉，必须持续点按维持高度
- **承诺**：玩家自己设的参考虚线，切歪了只能怪自己
- **苛刻**：评分系统不容忍单项失败，鼓励两项都做好
- **失败的戏剧性**：剪太短的瞬间不可逆，顾客的反应放大失败感

### 1.3 平台
HTML5 / Canvas 单页面 demo

---

## 二、核心玩法循环

```
NPC 入场
  ↓
NPC 用手比划想要的长度（含噪声 — 多次停留位置）
  ↓
玩家根据记忆设定一条水平虚线（承诺阶段）
  ↓
确认虚线，剪刀出现在刘海右上方
  ↓
玩家开始切割（剪刀从右向左推进）
  ↓
切割完成，结算评分（长度准确度 + 直线度）
  ↓
显示星级与顾客反应
  ↓
下一位顾客 / 重试
```

## 三、操作设计

### 3.1 输入
**两个键就够了：**

| 输入 | 作用 | 形式 |
|------|------|------|
| 抬升键 | 给剪刀向上的推力，对抗重力 | 按住持续生效 |
| 开合键 | 控制剪刀闭合切割 | 点按 |

建议绑定：
- 键盘：`空格` = 抬升，`J` 或鼠标左键 = 开合
- demo 阶段可调

### 3.2 剪刀旋转
**锁定旋转**。剪刀始终保持水平，不受任何力影响其角度。

### 3.3 水平移动（两个版本，需 demo 测试）

**A 版（首选）**：剪刀以恒定速度自动从右向左匀速移动。玩家只控制上下与开合。

**B 版（备选）**：每次开合动作时向左推进一段固定距离，不开合时水平静止。开合键同时承担"切割"和"推进"两个语义。

## 四、物理系统

### 4.1 剪刀刚体属性

| 属性 | 建议起手值 | 备注 |
|------|-----------|------|
| 重力 | 1.5~2 倍正常 | 紧迫感来源 |
| 线性阻尼 | 1.5~2.5 | 关键参数，决定手感稳定度 |
| 抬升推力 | 约 1.2~1.5 倍重力 | 按住缓升，松开缓降 |
| 旋转 | 锁定 | 始终水平 |

### 4.2 推力式手感
按住抬升键给一个向上的力（不是冲量），松开后重力接管。线性阻尼让速度不会无限累积。

预期玩家节奏：每秒 3~5 次按压，每次约 0.1 秒，近似悬停。
**节奏一被打断（比如按开合键），剪刀立刻开始下沉** —— 这是难度的核心来源。

### 4.3 头发切割逻辑
- 刘海被分为多束（建议 30 束左右），每束是垂直 segment
- 当剪刀闭合 **且** 刀刃 X 覆盖某束头发时，从剪刀当前 Y 高度切断该束
- 被切断的部分播放掉落动画后消失
- 记录每束被切断的 Y 坐标，用于直线度评分

## 五、NPC 比划机制

### 5.1 比划过程
NPC 入场后，手在脸侧上下移动若干次，含噪声：

| 类型 | 停留时长 | 视觉提示 |
|------|---------|---------|
| 干扰位置 | 0.3~0.8 秒 | 无 |
| 真正目标 | 1.0~1.5 秒 | 微小点头 + 停留更久 |

玩家需要从这一段动作里**自己提炼出真正的目标高度**。这模拟真实理发师"读懂顾客比划"的体验。

## 六、玩家辅助系统：参考虚线

### 6.1 设定阶段
NPC 比划完成后，进入"设定虚线"阶段：
- 玩家拖动一条水平虚线到屏幕中任意高度
- 玩家可以反复调整位置
- 点击"确认"按钮锁定，进入切割阶段

### 6.2 切割阶段
虚线**保留显示但变淡**（约 30% 透明度），作为切割时的视觉参考。
玩家任务：让剪刀沿着这条线水平切过去。

### 6.3 设计意图
- 把"记忆"和"执行"两个能力**干净地分开**
- 玩家自己设的线 = 玩家自己的承诺，失败的归因清晰
- 直线度评分基于实际切割轨迹的 max-min，与虚线位置无关，两个维度独立

## 七、评分系统

### 7.1 总分公式（乘法）

```
总分 = 长度分 × 直线分 × 100
```

每个分项归一化到 0~1。**短板效应**：任何一项接近 0，总分都接近 0。这种设计与游戏的 Foddy 风格一致。

### 7.2 长度分（带不对称惩罚）

```
偏差 = |玩家虚线高度 - NPC 真实期望高度|

if 玩家剪的比期望短:
    偏差 = 偏差 × 1.5    // 剪太短不可逆，加重惩罚

长度分 = exp(-(偏差 / 容差_长度)²)
```

容差初值：刘海总长度的 5%（具体看画布尺寸）

### 7.3 直线分

```
不平整度 = max(切割轨迹.Y) - min(切割轨迹.Y)
直线分 = exp(-(不平整度 / 容差_直线)²)
```

容差初值：8~10 像素

### 7.4 为什么用指数衰减而不是线性
- **奖励"几乎完美"**：偏差小的时候分数下降很慢，差一两像素仍能拿 95+
- **狠罚"灾难失误"**：偏差大的时候分数掉得飞快
- **中间区域有梯度**：每提升一点都能感觉到分数明显上涨

### 7.5 星级映射

| 总分区间 | 星级 |
|---------|------|
| 90~100 | ⭐⭐⭐ |
| 70~89 | ⭐⭐ |
| 50~69 | ⭐ |
| <50 | 重试提示 |

### 7.6 反馈呈现
- **大字**：星级（主反馈）
- **顾客表情动画**（情感反馈）
- **小字**（给硬核玩家）：长度偏差 X px / 不平整度 Y px

## 九、HTML Demo 关键参数（参考起手值）

```javascript
const CONFIG = {
  // 画布
  canvasWidth: 960,
  canvasHeight: 600,

  // 物理（单位：像素 / 秒）
  gravity: 1500,           // px/s² 向下
  thrustForce: 2200,       // px/s² 向上（按住时）
  linearDrag: 2.0,         // 线性阻尼系数（速度衰减）

  // 推进 — A 版自动
  autoAdvanceSpeed: 80,    // px/s 向左
  // 推进 — B 版每次开合
  advancePerCut: 12,       // px/次

  // 头发
  hairCount: 30,           // 刘海被切分的束数
  hairLeftX: 350,          // 刘海左边界
  hairRightX: 600,         // 刘海右边界
  hairTopY: 200,           // 头顶 Y 坐标
  hairBottomY: 350,        // 头发原始最低点

  // 评分容差
  lengthTolerance: 12,     // px
  straightTolerance: 10,   // px
  overCutPenalty: 1.5,     // 剪太短的额外惩罚倍数

  // NPC 比划
  totalGestureDuration: 4, // 总比划时长（秒）
  fakeStopRange: [0.3, 0.8],   // 干扰停留时长范围（秒）
  realStopRange: [1.0, 1.5],   // 真目标停留时长范围（秒）
  fakeStopCount: 2,            // 干扰停留次数
};
```

**所有数值都需要在 demo 阶段反复调试**，特别是 `linearDrag` 和 `thrustForce` 的比例，决定整个游戏的手感。

## 十、开发优先级

按这个顺序做原型，每一步做出来玩两把再决定是否继续：

1. **剪刀手感**（必须先打磨好，手感不对一切都白搭）
   - 重力 + 推力 + 阻尼调到能"啵啵啵"近似悬停
2. **头发切割机制**
   - 剪刀闭合时正确切断对应位置的头发
3. **虚线设定流程**
   - 拖动设线 + 确认 + 锁定
4. **基础评分**
   - 长度分 + 直线分 + 总分显示
5. **NPC 比划机制**
6. **顾客反应动画**
7. **关卡 / 多顾客**

## 十一、待定 / 未来扩展

- 多关卡 / 多顾客（每位顾客比划长度不同）
- 顾客类型差异（耐心、易动、奇怪偏好）
- 后片头发处理（目前只剪刘海）
- 失败状态：剪刀掉到底 → 顾客生气走人
- 道具系统：梳子（固定头发）、定型水（暂时增加阻尼）
- 时限模式 / 无尽模式
- 排行榜

## 设计原则备忘

> 写在最上面提醒自己

1. **难度从机制涌现，不靠外部惩罚硬加**
2. **玩家的失败感要"个人化"** —— 是我搞砸了，不是游戏在为难我
3. **苛刻的评分本身就是叙事** —— 不要装狠
4. **先把剪刀手感打磨好，其他都是后话**
